Berikut ada beberapa hal yang perlu diperhatikan dalam forensik OS Windows :

Deleted Data

Delete data Cuma kasih tau ke komputer kalo space yg dipake si file tsb available jika komputer emang butuh mau pake. Data yg ter-delete ini akan tetap bertahan sampe ada file lain yg overwrite dirinya.


File Carving

Metode ini dipakai utk mengekstrak data yg ada dalam sebuah unallocated space dalam hard drive. File2 tsb bisa saja punya karakteristik unik. (Header dan footer sebuah file)


Registry

Registry ini ibarat kata adalah database dari file2 konfigurasi sistem (kek sistem saraf manusia aja). Registry ini di dalamnya ada user config dan system config, juga preferences. Tapi Registry sangat potensial menjadi evidence.


MRU - Most Recent Used

Ini bisa jadi tempat yg potensial utk nyari evidence. File2 apa aja yg baru dibuka, dan lokasinya tercatat disini.


Jenis-jenis sleep

  • Sleep Mode Tujuannya untuk menghemat energi tetapi juga bertujuan untuk mengembalikan komputer ke operasi secepat mungkin

  • Hibernation juga merupakan mode hemat daya tetapi lebih ditujukan untuk laptop Ketimbang PC desktop. Disinilah kita mulai melihat beberapa manfaat investigasi yang potensial. Dalam mode ini, semua data dalam RAM ditulis ke hard drive, yang, seperti kita tahu, jauh lebih sulit untuk dihilangkan.

  • Hybrid Sleep adalah perpaduan dari dua mode sebelumnya dan ini ditujukan terutama untuk PC desktop. Teknik ini membuat sejumlah kecil daya listrik diterapkan ke RAM (utk menjaga data dan aplikasi ) dan menulis data ke disk


    • Jenis-jenis Metadata

  • Metadata deskriptif menggambarkan resource dalam arti penemuan (invention) dan identifikasi. Tipe ini dapat mencakup elemen-elemen seperti judul, abstraksi, author file , dan keywords.

  • Metadata struktural menunjukkan bagaimana collection dari object disusun bersama menjadi satu, seperti bagaimana halaman disusun untuk membentuk sebuah bab.

  • Metadata administratif menyediakan informasi untuk membantu mengelola resource, seperti kapan dan bagaimana informasi dibuat, jenis dokumen dan informasi teknis lainnya, dan siapa yang dapat mengaksesnya.


    • Komponen dalam Metadata

  • Created — date / time stamp yang dibuat sering menunjukkan kapan suatu file atau folder dibuat pada media tertentu , seperti hard drive. Bagaimana file itu sampai di sana dan membuat perbedaan. Pada umumnya,sebuah file dapat berupa disimpan , disalin, dipotong dan disisipkan, atau drag ‘n drop.

  • Modified — Tanggal dan waktu yang diubah ketika file sedang diubah dengan cara apa pun dan kemudian disimpan

  • Accessed — date / time stamp diperbarui setiap kali file diakses oleh file sistem. Diakses tidak berarti sama dengan dibuka scr langsung.


    • Recovery of Evidence

  • Restore Point adalah snapshot dari pengaturan dan konfigurasi sistem utama yg ada pada beberapa waktu yg spesifik. Snapshots ini dapat digunakan untuk mengembalikan sistem ke working order sebelumnya. Restore point dibuat dengan cara yang berbeda, dimana dapat dibuat oleh sistem secara otomatis sebelum peristiwa sistem utama, seperti saat sedang menginstal software. Restore point dapat pula dijadwalkan secara berkala.

  • Shadow Copies memberikan source data utk restore point. Dengan memakai shadow copy maka kita bisa melakukan demonstrasi bagaimana sebuah file bisa berubah dari waktu ke waktu. Bisa juga buat nahan file kopian dari file asli yg udh kehapus.
