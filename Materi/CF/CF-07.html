Berikut ini adalah beberapa hal yang harus kita periksa dalam forensik Internet dan email :


Peer To Peer / P2P

Peer to peer / P2P biasanya dipakai utk sharing file. Biasanya hal yg paling sering ditemui dalam trafik P2P adalah file2 bajakan. P2P sendiri bekerja dengan role sbg client juga sekaligus sebagai server.
Contoh P2P : Gnutella


Index.dat

Merupakan file binary, dan perannya sbg container. Biasa dipakai oleh IE. Index.dat isinya adalah informasi seputar URL yg kita kunjungi, berapa jumlah visit ke URL tsb, dan file ini sifatnya hidden dari user dan harus di-view memakai tools. Disini IE punya 3 direktori : history, cookie, dan temporary internet file. Karena itu index.dat berguna utk track informasi dan konten dari tiap direktor


NTUSER.dat

Isi file ini adalah preference setting dan informasi dari user. Ada 1 NTUSER.dat utk 1 user. Secara teknis dari registry, NTUSER.dat udah ada di folder user,


Cookie

File text kecil yang biasanya digunakan utk beragam keperluan, semisal melacak sesi , mengingat preferensi user utk bbrp web tertentu, isinya biasa juga berupa identitas kita juga.


Cache

Cache adalah file-file temporary yg biasa digunakan utk memangkas waktu download konten. Jadi cache ini konsepnya kek pake komponen sisa yg ada sebelumnya daripada download ulang lagi seluruh konten.


Caching & HTTPS

Yang perlu diingat adalah HTTPS web pages ga akan di-cache. Jadi sia2 kalo nyari cache HTTPS. (jika msh pake IE)


Internet History

Hal ini penting utk dicari, karena kita bisa lacak URL-URL mana saja yg terlibat dalam case yg kita tangani sekarang. History Browser ada biar kita gausah ketik full udh ada autocomplete.


Email

Biasanya evidence-evidence bisa juga ditemukan dari email-email suspek. Kadang orang suka simpen draft email-email gitu juga tuh dgn asumsi yg lain ga bakal baca drafnya. Bisa juga ada email malicious, dgn intention phishing dst.

Ada pula praktik 'Shared Email Account' dimana dalam praktiknya, email-email tersebut bisa dibuat siapapun dan disimpan ke dalam draft, dan bisa dihapus kapanpun kita mau, yang artinya teknik ini efektif untuk menghilangkan barang bukti.


Protokol Email ada 3, yaitu :

  • SMTP – simple mail protocol

    Utk kirim e-mail dari client, server mail tugasnya send and receive mail


  • POP – Post Office Protocol

    Dipakai email client utk receive pesan aja


  • IMAP – Internet Message Access Protocol

    Protokol utk komunikasi 2 arah yg dipakai client utk mengakses email dari server


    • Hal-hal yang kita bisa dapat dengan melakukan forensik pada email adalah :

  • Communication yg relevan ama case
  • Email address pelaku dan victim
  • IP address
  • Date / time


    • Lokasi terbaik untuk melakukan forensik email adalah sebagai berikut :

  • Suspect machine
  • Recipient machine
  • Company server / backup media
  • Smartphone
  • ISP
  • Server-server yg dilewati message yg kita lagi forensik (Mail Servers)
